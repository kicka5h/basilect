Self-service Azure IaC,
built for platform teams.

Basilect gives platform teams an opinionated, end-to-end framework for self-service Azure infrastructure. Product teams write a CDKTF stack in any of five languages — the platform owns everything beneath: Terraform modules, CI/CD pipelines, OPA policies, Azure Policy governance, state backends, and service principals.

Multi-language constructs

Python, TypeScript, C#, Java, and Go. Product teams pick their language; the platform team maintains one set of modules.

Layered enforcement

OPA policies gate every plan. Azure Policy enforces at the ARM level. Management locks protect stateful resources.

Zero static credentials

All authentication uses OIDC workload identity federation. No secrets stored in CI.

1
Developer writes a stack
A product team developer installs the construct library and writes infrastructure as code in their language of choice. No HCL required.
main.py
from basilect_infra import BaseAzureStack
from basilect_infra.constructs import (
    NetworkConstruct, DatabaseConstruct, SubnetConfig
)

class PortalStack(BaseAzureStack):
    def __init__(self, scope, id):
        super().__init__(scope, id,
            project="portal",
            environment="dev",
            location="eastus",
        )

        net = NetworkConstruct(self, "network",
            project="portal",
            environment="dev",
            resource_group="portal-dev-rg",
            location="eastus",
            address_space=["10.0.0.0/16"],
            subnets={"app": SubnetConfig("10.0.1.0/24")},
        )

        DatabaseConstruct(self, "db",
            project="portal",
            environment="dev",
            resource_group="portal-dev-rg",
            location="eastus",
            subnet_id=net.subnet_ids["app"],
            dns_zone_id=net.dns_zone_ids["postgres"],
            admin_password=os.environ["DB_PASSWORD"],
        )
2
PR triggers validation
The developer opens a pull request. The reusable workflow pipeline kicks in automatically — formatting, validation, and change detection.
.github/workflows/infra.yml
name: Infrastructure
on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  validate:
    uses: basilect/reusable-workflows/.github/workflows/tf-validate.yml@v1.4.0

  changes:
    if: github.event_name == 'pull_request'
    uses: basilect/reusable-workflows/.github/workflows/tf-changes.yml@v1.4.0

  plan:
    needs: [validate, changes]
    if: github.event_name == 'pull_request'
    uses: basilect/reusable-workflows/.github/workflows/tf-plan.yml@v1.4.0
    with:
      environments: ${{ needs.changes.outputs.affected }}
3
OPA policies gate the plan
Every terraform plan is evaluated against the full OPA policy suite. Public resources, unauthorized network changes, missing tags — all blocked before apply.
policy/deny_public_resources.rego
package basilect.deny

import rego.v1

deny contains msg if {
    some resource in input.resource_changes
    resource.change.after.public_network_access_enabled == true
    msg := sprintf(
        "Resource '%s' enables public network access. "
        "All resources must use private endpoints.",
        [resource.address]
    )
}

deny contains msg if {
    some resource in input.resource_changes
    resource.type == "azurerm_public_ip"
    msg := sprintf(
        "Resource '%s' creates a public IP. "
        "Public IPs are not allowed in any environment.",
        [resource.address]
    )
}

# ✓ Plan passed — 0 violations
